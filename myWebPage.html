¿Cuál es la diferencia entre Internet y la World Wide Web?

  • El internet es la infraestructura interconectada de diferentes computadoras y ordenadores. Es una 'red de redes' (podría decirse que la red de redes). Sienta las bases para la Web.
  • En los 60's ARPA (Advanced Research Project Agency) del departamento de defensa de USA financió una investigación para desarrollar protocolos que permitieran que las computadoras se interconectaran y eventualmente se comunicaran. Así se creo la primera 'red' de investigación para este proyecto. Conectaba centros de investigación, universidades y demás. Pero al mismo tiempo se desarrollaban otras redes, con otros protocolos. Pero luego se quisieron conectar estas redes, pero hubo problemas porque todas tenían diferentes protocolos. Así nació TCP/IP.
  • Internet == Inter Network
  • Una network (o red) es una serie de sistemas de computo interdonectados para comunicarse e intercambiar datos. Son nodos conectados vía cables o vías inalámbricas. El internet entonces es la red de redes global.
  • Muchos servicios se han creado para utilizar del internet. Uno de ellos la World Wide Web. El servicio más comunmente usado en el internet. Share, get and search data. Un sistema que interconecta recursos en el internet. Esto mediante hipervínculos, referenciados con URL's. Se usan buscadores para acceder a los recursos de la web, y estos son presentados usualemente en web pages conglomeradas en websites.

¿Partes de un URL?

Son diez, y aunque no contiene todas estas partes a la vez, mínimo siempre conteine tres:

  1. scheme (esquema): protocolo para accesar a una página web. Usualmente https (Hyper Text Transfer Protocol Secure)
  2. subdomain (sub-dominio)
  3. top-level domain (dominio de nivel superior)
  4. second-level domain (dominio de segundo nivel)
  5. subdirectory (sub directorio)
  6. parameter (parámetro)
  7. port (puerto)
  8. path (camino)
  9. query (consulta)
  10. fragment (fragmento)

¿Cuál es el propósito de los métodos HTTP: GET, HEAD, POST, PUT, PATCH, DELETE?

  • GET: Recupera datos mediante una solicitud, más no los incluye. Representaciones de recurso.
  • HEAD: Accesa solo a los headers sin el cuerpo en el request del URL. Se usa para obtener tamaños de archivo por ejemplo.
  • POST: Envía una serie de datos al servidor. Se diferencia con PUT ya que las llamadas subsecuentes pueden tener efectos diferentes. 'Enviar a una entidad un recurso específico'.
  • PUT: (idempotente)Crea un elemento o cambia uno existente con las datos de la petición generada.
  • PATCH: Se utiliza modificar parcialmente elementos. Un parche literalmente. Es parcial a difernecia de PUT y no es idempotente, pero puede serlo.
  • DELETE: Elimina por completo los recursos especfíficados.

¿Es responsabilidad del desarrollador corregir un sitio web si un usuario reporta que intentó acceder al sitio y se encontró con un error 404? ¿Por qué?

El error 404 puede suceder por varias razones, y muchas veces el usuario puede arreglar el error y acceder a la información que busca. Por ello no es necesariamente responsabilidad de un desarrollador atender esto.

¿Es responsabilidad del desarrollador corregir un sitio web si un usuario reporta que intentó acceder al sitio y se encontró con un error 500? ¿Por qué?

Hay un error en el servidor interno, por alguna razón no se puede accesar. Es responsabilidad del desarrollador resovler esto.

¿Cuáles son las diferencias principales entre HTML 4 y HTML5?

HTML 5 contiene nuevos elementos para mejorar la estructura y la semántica de este lenguaje. La declaración de un documento de HTML se hizo más sencilla y menos tediosa que en HTML 4. También ahora se tiene una mayor cantidad de API's para HTML 5. La naturaleza de esta nueva variante también generó una mejor forma de estructurar estos documentos.

¿Qué componentes de estructura y estilo tiene una tabla?

La table se construye con 'table'. Cosnta de un head o encabezado (th o thead), un cuerpo (tbody) construido con 'td' y un footer (tfoot). Todas las celdas construidas con 'td'.

¿Cuál es el ciclo de vida de los sistemas de información? y ¿Cuál es el ciclo de desarrollo de sistemas de información?

El ciclo de vida se refiere al proceso completo (en el paso del tiempo) desde la planeación, el análisis, diseño e implementación de un sistema. Pasando hasta su uso, funcionamiento y mentenimeitno. Mientras que el ciclo de desarrollo sólo se refiere al plazo de tiempo que toma la creación del sistema planteado.
